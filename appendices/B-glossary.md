# 附录 B：术语表

| 术语 | 本书口径 |
|---|---|
| FDE | 进入客户或业务现场，把需求、流程、技术和交付责任连接起来的落地角色，不限定具体职位名称 |
| 六类结构要素与反馈回路 | 本书定义的项目诊断框架：人、流程、数据、工具、规则、指标六类结构要素，加一条把运行证据带回责任、流程和知识更新的反馈回路 |
| 四平面 | 本书定义的业务、解决方案、平台和控制四个责任视图；不是外部标准或厂商产品分层 |
| 普通软件/规则 | 用确定代码、规则或状态机稳定执行任务的系统能力 |
| RAG | 检索增强生成；在生成前取得外部知识，同时需要处理权限、引用、时效和评估 |
| 工作流 | 按预先设计的步骤、状态、分支、审批和异常执行的流程 |
| Agent | 能围绕目标选择路径或工具的系统组件；自主性必须有权限、预算和停止边界 |
| 上下文工程 | 为一次任务选择、组织、压缩和治理用户、知识、工具、规则与历史信息 |
| 模型网关 | 统一模型调用身份、渠道、路由、配额、成本、审计和降级的控制入口 |
| 本地推理 | 在企业批准的自有或隔离环境提供模型推理；不自动等于安全或生产可用 |
| 混合架构 | 同一业务流程按任务和数据使用多种批准部署与模型路径 |
| 人审 | 人类对事实、业务授权、风险或异常承担具体审核责任，不只是形式确认 |
| 自主性等级 | 本书用于逐项分配动作权限的 A0 禁止、A1 建议、A2 草拟、A3 确认后执行、A4 边界内自动执行；不是企业成熟度等级或外部标准 |
| 评估集 | 版本化的输入、上下文、期望、禁止行为、评分和负责人集合 |
| 发布门禁 | 变更或版本进入下一环境前必须满足的阻断、非退化和业务条件 |
| LLMOps | 围绕模型、Prompt、知识、评估、追踪、成本和回归的运行机制 |
| AI SRE | 将服务目标、可靠性、告警、降级、回滚和事故响应应用到 AI 系统 |
| Trace | 串联用户、检索、模型、工作流、工具、人审和业务结果的一次请求轨迹 |
| SLI/SLO | 服务水平指标及其目标窗口，用于定义质量、风险、性能、可靠性和成本要求 |
| ADR | 架构决策记录，保存背景、候选、选择、取舍、假设和重新评审条件 |
| TCO | 总拥有成本，包含建设、模型/基础设施、运维、安全、评估、支持和退出成本 |
| 单次成功任务成本 | 总成本除以满足质量、风险和业务终点的任务数量 |
| POC | 用有限范围和真实证据验证关键假设，不等于功能演示 |
| MCP | Model Context Protocol；用于连接 AI 应用与工具、资源和提示的开放协议，生产使用需按版本核查授权与安全要求 |
| Saga | 把跨系统长事务拆成可提交步骤，并为已完成步骤定义补偿动作的事务模式 |
| 幂等键 | 标识同一业务意图的稳定键；重复请求只能复用同一结果，载荷冲突必须被拒绝 |
| 结果未知 | 调用方无法确定外部动作是否已发生的中间状态，需要通过业务后置条件对账，不能直接重试 |
| 背压 | 下游容量不足时限制、排队或拒绝上游负载，避免局部拥塞演变为级联故障 |
| TTFT | Time to First Token，首个输出 token 的等待时间，反映交互式推理的首响体验 |
| KV Cache | 推理时复用注意力键值状态的缓存；会占用显存，并影响并发与容量规划 |
| 语义缓存 | 按语义相似度复用已验证结果的缓存；必须处理权限、时效、版本和敏感信息边界 |
| RACI | Responsible、Accountable、Consulted、Informed 的责任分配方法 |
| IAM | 身份与访问管理；覆盖主体身份、认证、授权、凭据和生命周期控制 |
| DLP | 数据防泄漏控制，用于识别和阻止敏感数据的不当传输或使用 |
| SBOM | 软件物料清单，用于记录组件、版本和依赖，支持供应链风险与漏洞处置 |
| 提示注入 | 不可信内容试图改变系统指令、越权调用工具或泄露数据的一类攻击与失效模式 |
| 错误预算 | 在 SLO 窗口内可容忍的失败额度，用于在可靠性与变更速度之间作出可执行取舍 |
| 金丝雀发布 | 先向小比例真实流量发布变更，以监测门禁和回滚条件后再逐步扩大 |
| 制品清单 | 记录一次发布包含的代码、配置、Prompt、知识、评估集、模型与校验和的版本化清单 |

## 易混淆术语

- 私有化部署不等同于桌面本地运行，也不等同于安全认证。
- 知识库不等同于文件上传或向量数据库。
- Agent 不等同于任何带聊天入口的 AI 应用。
- 工作流成功不等同于业务成功；需要检查业务后置条件。
- 高风险样本通过不等同于系统零风险。
- 有日志不等同于可观测；日志需要归属、指标、负责人和响应动作。
