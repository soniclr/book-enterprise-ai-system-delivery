# 公开范围说明

本仓库采用“公开正式源稿与私有编辑过程分离”的方式。正式 Markdown、配套资源和可复现构建入口进入公开 Git 历史；历史改稿和内部审读过程继续保留在私有工作目录。

## 适合公开

| 内容 | 公开方式 | 理由 |
|---|---|---|
| `manuscript/`、`cases/`、`appendices/` | Markdown 源稿 | 提供完整、可审阅和可提交勘误的正式文本源文件 |
| `assets/`、`publication/` | 插图、Graphviz 源文件与出版样式 | 保证 Markdown 引用完整，并保留出版构建所需资源 |
| `code/` | 源码与测试 | 仅含合成数据，无外部服务凭据，按 MIT License 开源 |
| `tools/` | Markdown 模板 | 通用、空白、无客户信息，便于学习和内部项目诊断 |
| `scripts/` 与构建依赖清单 | 校验和出版构建入口 | 支持从公开源稿重新生成 Markdown、DOCX、PDF 和附件 |
| `企业AI系统落地-v0.1.pdf` | 仅作为 GitHub Release 附件 | 固定版式发行物不进入 Git 历史，避免仓库体积随版本增长 |
| README、构建说明、变更记录、贡献指南、校验和 | Git 跟踪 | 公开协作与版本验证所需文档 |

## 不进入公开仓库

| 内容 | 原因 |
|---|---|
| DOCX、构建生成的合并版 Markdown、附件包和旧版成品 | 均可由正式源稿重新生成；直接跟踪会重复内容并模糊版本边界 |
| `archive/` 中的历史改稿 | 含过时表达、重复内容和内部修订轨迹，不属于正式发行物 |
| `editorial/`、写作进度与用户反馈记录 | 属于内部创作、审读和工作过程，不面向外部读者 |
| 批量改写、语气调整和历史迁移脚本 | 直接修改书稿且只服务于内部编辑过程，不是正式构建链的一部分 |
| `.DS_Store`、`__pycache__` 等缓存 | 本机生成文件，没有发行价值 |

## 许可边界

“可以在公开仓库下载”不等于“所有内容均采用开源许可证”。代码与书稿内容分别适用不同条款，详见根目录的两个许可文件。
